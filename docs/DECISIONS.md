# Surface Anomaly Detection Technical Decisions (ADR)

이 문서는 표면 이상 탐지 프로젝트(SG_proj_005_sad)의 구조 설계 및 라이브러리 채택 과정에서 내려진 중요 아키텍처 결정 사항(Architecture Decision Records)을 보존합니다.

---

## 1. ADR 01: OpenCV(CV2) 의존성의 전면 제거 및 PIL/Numpy 기반 재구현

### 상태
승인됨 (Accepted)

### 컨텍스트
과거 버전에서는 이미지 전처리, 색공간 변환(BGR/RGB) 및 아노말리 히트맵 오버레이 시각화를 위해 대중적인 OpenCV 라이브러리(import cv2)를 활용하였습니다. 
그러나 Streamlit Cloud 등의 클라우드 샌드박스 배포 환경(Debian 기반)에 본 시스템을 릴리스할 때, OS 레벨의 그래픽스 공유 라이브러리(libGL.so.1, libglib-2.0.0 등)가 누락되어 애플리케이션의 import 단계부터 크래시가 발생하는 장애가 지속되었습니다. `opencv-python-headless` 설치나 `packages.txt` 구성을 통한 보완을 시도하였으나, 배포 머신의 런타임 업데이트에 따라 안정성이 보장되지 않는 근본적 문제가 존재했습니다.

### 의사 결정
추론 및 웹 대시보드 구동 흐름 전체에서 OpenCV 의존성을 100% 완전 제거하기로 결정하였습니다. 
오버레이 투명도 계산, 히트맵 리사이즈, 컬러맵 렌더링 등의 모든 핵심 연산은 순수 NumPy 연산과 Python Imaging Library (PIL), 그리고 Matplotlib Colormap Utility만을 사용하여 In-memory 상에서 구현합니다. Anomalib 내부 서브모듈의 간접적인 cv2 임포트 트리거는 app.py 기동 초기 단계에서 unittest.mock.MagicMock 기법을 동적으로 적용하여 강제로 인메모리 모킹(In-memory mocking) 처리해 원천 봉쇄합니다.

### 결과
- 배포 무결성 확보: 리눅스 OS 레벨의 그래픽 패키지 의존성을 완전히 제거함으로써, Streamlit Cloud, Docker CPU 환경 등 그 어떤 경량 배포 환경에서도 설치 즉시 무결 기동되는 탁월한 호환성을 확보했습니다.
- 성능 정합성: 리사이즈 연산 시 PIL의 BICUBIC 보간법을 사용하여 OpenCV 기반의 보간 대비 이상 탐지 점수 및 좌표 추출 오차가 발생하지 않음을 철저히 검증하였습니다.

## 2. ADR 02: SAM2(Segment Anything Model 2) 연동 및 Peak-point 유도 프롬프트 도입

### 상태
승인됨 (Accepted)

### 컨텍스트
PatchCore 알고리즘은 소량의 데이터로 고해상도 이상 유무를 판정하는 데 매우 강력하지만, 아웃풋으로 도출되는 anomaly_map(이상 감도 맵)은 가우시안 평활화가 적용된 부드러운 히트맵 형태를 취합니다. 이로 인해 미세 결함의 실제 기하학적 형태나 결함의 물리적 면적(제곱밀리미터 등)을 정확히 산출하거나, 날카로운 실제 결함의 외곽선을 포착하여 시각화하는 데 한계가 있었습니다.

### 의사 결정
이상 히트맵의 최댓값을 바탕으로 최고 위험 좌표(Peak-point)를 수학적으로 산출하고, 이를 다용도 제로샷 이미지 분할 엔진인 SAM2(Hiera-Tiny)의 양의 점 프롬프트(Positive point prompt)로 주입하는 하이브리드 파이프라인을 구축하기로 결정하였습니다. 
PatchCore가 "결함의 대략적인 핵심 중심점"을 찾고, SAM2가 그 점으로부터 "정확한 물리적 결함 경계선"을 픽셀 정밀도로 확장 및 마스킹하도록 연계 구조를 수립합니다.

### 결과
- 결함 면적 분석 가능: 모호했던 히트맵의 붉은 점을 넘어, 결함 경계를 선명하게 추출함으로써 이상 부위의 픽셀 면적 계측이 기술적으로 가능해졌습니다.
- 극대화된 시각화 퀄리티: 대시보드 화면상에서 원본 이미지 위에 칼날 같이 정밀한 결함 경계 마스크가 오버레이 렌더링되어, 사용자에게 극적인 신뢰도와 분석 체감 성능을 제공합니다.
